When you've found an issue with MyBB, head over to the development site, log in and then click on the "New Issue" tab.

There are lots of options that you can select here, but you'll only need a few of them. Here are a few guidelines about what to do when you report a new issue.

  • Enter a descriptive subject for the issue. Try not to be vague or too ambitious - creating an issue such as Javascript issue isn't great, but writing Javascript issue when creating a new user is pretty much ideal. Writing Javascript issue line 264 general.js in the acp when creating a new user is a bit too much detail(!). The best subject title will describe the problem in a short sentence.
  • When entering the description of your problem, try to give us as much information as you can. If you can remember how you found the issue (reproduction steps) then please include that too. There is a file upload box too where you can add screenshots if you can.

    If your problem includes an error message, whether it's something on screen or an SQL error, then please include that too.
  • The next step is to select a category for the issue. This is normally where you found the problem. Select an area from the list, and if it isn't there, put it as other.
  • Set the Reproducibility - tell us how often you find this problem.
  • Set the Reported In MyBB Version - tell us which version of MyBB this problem affects. It affects all versions, select the latest one.

That's all there is to it. Please do not set assignee or target versions, start or due dates and estimated time as these are for the MyBB Team to choose from. Setting these by yourself can mess around with our roadmaps and targets!

If you can provide more details, such as browser, PHP details and database information then please include it - especially if there is an javascript, PHP warning or SQL error in your problem.
